Easy Batch FAQ

1. Can I use Bean Validation (JSR 303) with Easy Batch?

Yes. Easy Batch uses the reference implementation Hibernate validator to validate domain objects.

The following steps describe how to use Bean Validation API with Easy Batch :

  • Add the easybatch-validation module to your dependencies : 

    <dependency>
    <groupId>io.github.benas</groupId>
    <artifactId>easybatch-validation</artifactId>
    <version>${version}</version>
</dependency>

  • Annotate your POJO with Bean Validation API annotations
  • Register the BeanValidationRecordValidator using the EasyBatchEngineBuilder : 

    EasyBatchEngine easyBatchEngine = new EasyBatchEngineBuilder()
    .registerRecordValidator(new BeanValidationRecordValidator<YourPojoType>())
    .build();

2. Does Easy Batch implement the Batch Processing for Java Platform (JSR 352) ?

No because the JSR 352 is pretty new and Easy Batch does not implement it yet. This is currently being analysed and Easy Batch will probably implement this JSR because it is already designed to split the batch into multiple steps as in JSR 352.

3. Can I monitor Easy Batch execution with JMX?

Yes. By default, Easy Batch enables a JMX MBean named io.github.benas.easybatch.core.jmx:type=EasyBatchMonitorMBean at startup. You can use any standard JMX compliant tool to monitor Easy Batch metrics. For more details about this feature, please refer to the JMX section.

4. Can I use Easy Batch with Spring ?

Yes. You do not have to write any specific code to use Easy Batch with Spring. You can configure your batch components in Spring and then register them within Easy Batch engine, that's it! You can find a complete example of using Easy Batch with Spring here.

5. How can configure Easy Batch logging ?

Easy Batch framework core uses the java.util.logging API. By default, Easy Batch logs messages to the standard output. Please refer to official documentation of the the java.util.logging API for more details about how to customize it.
