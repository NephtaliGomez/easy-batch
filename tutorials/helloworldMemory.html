Hello World In-Memory Data Source Tutorial

1. Description

This tutorial is the same as the Hello World CSV tutorial except that greetings are read from a String in memory rather than a CSV flat file. We will process a String containing greeting records and generate greeting messages. To keep the tutorial simple, the input String has the following content:

1,foo
2,bar

Each record is composed of 2 fields :

  • an id
  • a person name

For each record, the batch application should print to the console a greeting of the format : Hi foo! Your id is 1 .

Easy Batch development is POJO-based, so we would like to map each String record to an instance of the following Greeting bean:

public class Greeting {

    private int id;

    private String name;

    // Getters and Setters omitted

    public String getGreetingMessage() {
        return "Hi " + name + "! Your id is " + id;
    }
}

The id and name fields in String records should be mapped respectively to the id and name fields in the Greeting bean.

You will see that all these requirements can be easily implemented using Easy Batch, so let's get started!

2. Implementation using Easy Batch

The goal of Easy Batch is to let you concentrate on your batch processing logic and to handle for you all the plumbing code of reading, filtering, parsing and mapping records to domain objects.

First, let's implement our batch processing logic by extending the AbstractRecordProcessor class:

public class GreetingProcessor extends AbstractRecordProcessor<Greeting> {
    @Override
    public void processRecord(Greeting greeting) throws Exception {
        System.out.println(greeting.getGreetingMessage());
    }
}

This processor will print the greeting message for each input record as required.

Actually, we have almost finished our batch implementation! We should now configure Easy Batch engine to :

  1. Read records from the input String
  2. Map each record to an instance of the Greeting bean
  3. Process each record using our GreetingProcessor

The following listing provides this configuration:

public class Launcher {

    public static void main(String[] args) throws Exception {

        // Create the String data source
        String dataSource =
                "1,foo\n" +
                "2,bar";

        // Build an easy batch engine
        EasyBatchEngine easyBatchEngine = new EasyBatchEngineBuilder()
            .registerRecordReader(new StringRecordReader(dataSource))
            .registerRecordMapper(new DelimitedRecordMapper(Greeting.class, new String[]{"id", "name"}))
            .registerRecordProcessor(new GreetingProcessor())
            .build();

        // Run easy batch engine
        EasyBatchReport easyBatchReport = easyBatchEngine.call();

        // Print the batch execution report
        System.out.println(easyBatchReport);
    }
}

That's all, we can now run the batch and see the results.

3. Running the batch

The complete source code of this tutorial is available here.

To run the tutorial with maven, launch the following command in the easybatch-tutorials directory :

mvn exec:java -PrunHelloWorldInMemoryTutorial

If everything is ok, you should see something like the following output in the console :

INFO: Initializing easy batch engine
INFO: Data source: In-Memory
INFO: Strict mode: false
INFO: Total records = 2
INFO: easy batch engine is running...
Hi foo! Your id is 1
Hi bar! Your id is 2
INFO: Shutting down easy batch engine

As expected, Easy Batch generated a greeting message for each person.

4. Conclusion

In this tutorial, you have learned how to process data from an in-memory data source using Easy Batch.

With Easy Batch, all we have done is :

  • Implementing the business logic of our batch
  • Providing configuration meta data

Easy Batch handled all the plumbing code of reading, filtering, parsing and mapping String records to instances of our domain object Greeting and actually has reduced this plumbing code from hundred lines to only a very few lines! (listing above)

The following table summarizes what we have done and what Easy Batch has done for us :

Task You Easy Batch
Provide configuration meta data
Provide record processing logic
Read input String record by record
Map records to domain objects
Apply business processing logic
